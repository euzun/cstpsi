# Benchmark Configs

JSON parameter files passed to `cstpsi_cli bench --params <file>`. Each pins a `(N, k, m, partition_size, dataset)` combination for one benchmark cell.

## Files

| Config | Rows | Use |
|--------|------|-----|
| `smoke_1k_N64_k2.json` | 1000 | Smoke test. Pairs with the `{enr,qry,expected}_1k_*.csv` files that `experiments/datagen.py` produces. Quickest sanity run. |
| `bench_1k.json` | 1000 | Full bench cell at 1K rows |
| `bench_10k.json` | 10000 | Full bench cell at 10K rows |
| `bench_100k.json` | 100000 | Full bench cell at 100K rows |

All four use `N=64`, `k=2`, `m=4096`, `partition_size=32` (the paper-default deployment shape). They differ only in `nrof_que_ids` and the dataset paths under `dataset.enrollment_path` / `query_path`.

## Schema

See `parameters/README.md` for the field-by-field schema. The configs here add no fields beyond that schema; they just pin specific values.

## Adding a new config

Copy `bench_10k.json`, adjust the row count and the three `dataset.*_path` strings, and point them at CSVs generated by `experiments/datagen.py`. Hand the resulting file to `cstpsi_cli bench --params <new>.json --output-jsonl run.jsonl` and validate with `scripts/validate_jsonl.py`.
