# Demo Scripts

End-to-end demonstrations of the CSTPSI protocol on small synthetic data. Two
modes are covered: a single-process CLI (`cstpsi_cli`) and a split
sender/receiver over a network socket (`cstpsi_sender` + `cstpsi_receiver`). The
network split is bit-for-bit equivalent to the in-process path; it adds only
transport overhead.

All demos use the CSTPSI integer-set input format (`id, item_0..item_{N-1},
lbl_0..lbl_{k-1}` for enrollment). The datasets are generated on the fly by
`experiments/datagen.py` (the same generator the evaluation harness uses), so
nothing large is committed. The default configuration is `parameters/demo_1k.json`
(D=1000 rows, N=64, 23-bit label, CSTPSI with T=2 token rounds).

## Prerequisites

Build the binaries first:

```bash
cmake -S .. -B ../build -DCMAKE_BUILD_TYPE=Release
cmake --build ../build -j
```

Demo data is generated automatically the first time each script runs.

## In-process demo

```bash
./demo_small.sh
```

What it does: generates D=1000 enrollments + 5 true-positive / 5 true-negative
queries (integer-set format), runs one full PSI pass via `cstpsi_cli csv_run`
against `parameters/demo_1k.json`, and reports that every true-positive query
recovered its match while no true-negative was falsely accepted (FRR=0, FAR=0).
Results are written to `results/demo_output.csv`. Runtime under a minute.
Override with env vars `D`, `TP`, `TN`, `BUILD_DIR`.

## Network-mode demos

Sender and receiver run as two cooperating binaries talking over ZeroMQ (default
port 1212): `cstpsi_sender` holds the database and serves the protocol;
`cstpsi_receiver` ships encrypted queries and recovers the labels.

| Script | Use |
|--------|-----|
| `demo_network.sh` | Automated: generates a small synthetic dataset, runs one CSTPSI cell over loopback, verifies recovery |
| `run_sender.sh` | Manual: start the sender in the foreground |
| `run_receiver.sh` | Manual: connect a receiver to `localhost` or a remote sender |
| `demo_split_screen.sh` | tmux side-by-side view of sender and receiver |

### Automated demo

```bash
./demo_network.sh
```

Generates a small synthetic dataset with `experiments/datagen.py` (default 1K
rows, `parameters/demo_1k.json`, CSTPSI T=2), runs one sender + receiver cell
over loopback via `experiments/lib/run_cell.sh`, then verifies the recovered
labels with `experiments/verify.py`. The per-cell result JSON is written to
`results/demo_cell.json`. Override via env vars `PORT`, `BUILD_DIR`, `TP`/`TN`,
`D`, `THREADS`.

### Manual two-terminal flow

Both manual scripts take two optional positional arguments (data CSV, params
JSON) and read the host/port from environment variables. They run the full
CSTPSI protocol (T=2 token rounds + 1 label chunk = 3 wire rounds).

Terminal 1:

```bash
./run_sender.sh                       # defaults: data/enr_1k_lbl23bit.csv, parameters/demo_1k.json
./run_sender.sh DB_CSV PARAMS_JSON    # override db and params
```

Terminal 2:

```bash
./run_receiver.sh                     # localhost
ADDR=192.168.1.100 ./run_receiver.sh  # remote sender (ADDR env var; default 127.0.0.1)
./run_receiver.sh QUERY_CSV PARAMS_JSON   # override query and params
```

Both default to `PORT=1212` (override with the `PORT` env var). The sender stays
up until Ctrl+C. The receiver processes every query, writes per-round outputs to
`results/rounds/`, and then runs `experiments/verify.py` to report FRR/FAR and
label recovery. If the enrollment/query CSVs are missing, `run_sender.sh`
generates them.

### Split-screen view

```bash
./demo_split_screen.sh
```

Opens both binaries in side-by-side tmux panes. Requires `tmux`
(`brew install tmux` on macOS).

### Configuration

The scripts default to `parameters/demo_1k.json` and the generated
`demo/data/enr_1k_lbl23bit.csv` / `qry_1k.csv`. Override the CSVs and the params
JSON via the two positional arguments, and the host/port via env vars:

```bash
PORT=2000              ./run_sender.sh    path/to/enr.csv  path/to/params.json
ADDR=host PORT=2000    ./run_receiver.sh  path/to/qry.csv  path/to/params.json
```

To use a larger configuration, pass any of the `experiments/configs/*.json`
files as the second argument and supply matching CSVs generated by
`experiments/datagen.py`. The underlying binaries also accept the
explicit `--dbFile`/`--queryFile`/`--paramsFile`/`--senderAddr`/`--port` and the
protocol flags `--mode`/`--nrofTokenRounds`/`--nrofLabelChunks`/`--nrofRounds`
if you invoke `cstpsi_sender` / `cstpsi_receiver` directly (see
`experiments/lib/run_cell.sh` for the exact invocation).

### Troubleshooting

- **`Address already in use`** on the sender: another sender is still bound to
  the port. `lsof -i :1212` then `kill <pid>`, or pick a free port with `PORT`.
- **Receiver cannot connect**: confirm the sender printed that it is listening,
  check the firewall, verify `ADDR` resolves (`ping host`).
- **`libemp-tool.dylib: image not found` (macOS)**: the scripts set
  `DYLD_LIBRARY_PATH` to `build/emp_install/lib`; if you invoke a binary
  directly, export it yourself.

For the wire format and message sequence see `docs/NETWORK_PROTOCOL.md`. Source:
`app/sender.cc`, `app/receiver.cc`, and the network layer under `src/`.

## Data generation

```bash
python3 ../experiments/datagen.py --sizes 1k --labels 23bit --tp 5 --tn 5 --output-dir data
```

This writes, into `data/`:

- `enr_<size>_lbl<label>.csv` -- enrollment: `id, item_0..item_{N-1}, lbl_0..lbl_{k-1}`
- `qry_<size>.csv` -- queries: `id, item_0..item_{N-1}, is_tp`
- `expected_<size>_lbl<label>.csv` -- true-positive ground-truth labels

See `experiments/datagen.py --help` for the full option set (sizes, label
widths, true-positive / true-negative counts, seeds).
